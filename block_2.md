### 1. Задание 1. Изоморфизмы

Нужно реализовать функцию, которая проверяет, являются ли две строки изоморфными.

Две строки считаются изоморфными, если:
- каждый символ первой строки всегда переходит в один и тот же символ второй строки;
- два разных символа первой строки не могут переходить в один и тот же символ второй строки;
- порядок символов сохраняется.

Удобно решать задачу с помощью двух словарей:  
- первый хранит соответствие символов из `s` в `t`;  
- второй — обратное соответствие символов из `t` в `s`.

Сначала проверяем, что длины строк одинаковые.  
Если длины разные, строки сразу не могут быть изоморфными.

Дальше идем по символам двух строк одновременно:  
- если символ уже встречался, проверяем, что соответствие не нарушилось;  
- если символ новый, записываем новое соответствие в оба словаря.

Если на всем проходе противоречий не возникло, строки изоморфны.

Текст кода:

    def is_isomorphic(s, t):
        if len(s) != len(t):
            return False

        map_s_to_t = {}
        map_t_to_s = {}

        for char_s, char_t in zip(s, t):
            if char_s in map_s_to_t:
                if map_s_to_t[char_s] != char_t:
                    return False
            else:
                map_s_to_t[char_s] = char_t

            if char_t in map_t_to_s:
                if map_t_to_s[char_t] != char_s:
                    return False
            else:
                map_t_to_s[char_t] = char_s

        return True


    s = 'paper'
    t = 'title'
    print(is_isomorphic(s, t))

Вывод:

`True`

Оценка оптимальности:  
- время: `O(n)`, где `n` — длина строки, потому что делается один проход;  
- память: `O(n)`, так как используются два словаря для хранения соответствий.

Решение является оптимальным по времени, потому что проверить строки быстрее чем за один проход нельзя.

### 2. Задание 2. Натуральная последовательность

Нужно найти одно отсутствующее число в последовательности натуральных чисел от `1` до `n`.

Удобное решение — использовать сумму первых `n` натуральных чисел:

`1 + 2 + ... + n = n * (n + 1) / 2`

Тогда:  
- находим ожидаемую сумму чисел от `1` до `n`;  
- находим фактическую сумму элементов массива;  
- разность между ними и будет пропущенным числом.

Чтобы определить `n`, можно взять длину массива и прибавить `1`, так как одного числа не хватает.

Текст кода:

    def missing_number(nums):
        n = len(nums) + 1
        expected_sum = n * (n + 1) // 2
        actual_sum = sum(nums)
        return expected_sum - actual_sum


    nums = [1, 2, 3, 4, 5, 6, 8, 9, 10, 11]
    print(missing_number(nums))

Вывод:

`7`

Оценка оптимальности:  
- время: `O(n)`, потому что нужно один раз пройти по массиву для подсчета суммы;  
- память: `O(1)`, так как используются только несколько переменных.

Это решение оптимально по памяти и является одним из самых удобных по реализации.

### 3. Задание 3. Факторизация

Нужно разложить натуральное число на простые множители.

Идея решения такая:  
- сначала делим число на `2`, пока оно делится;  
- затем проверяем только нечетные делители, начиная с `3`;  
- нет смысла проверять делители больше квадратного корня из числа;  
- если после всех делений осталось число больше `1`, значит оно тоже простое и его нужно добавить в ответ.

Текст кода:

    def prime_factors(n):
        factors = []

        while n % 2 == 0:
            factors.append(2)
            n //= 2

        divisor = 3
        while divisor * divisor <= n:
            while n % divisor == 0:
                factors.append(divisor)
                n //= divisor
            divisor += 2

        if n > 1:
            factors.append(n)

        return factors


    n = 56
    print(prime_factors(n))

Вывод:

`[2, 2, 2, 7]`

Оценка оптимальности:  
- время: `O(sqrt(n))` в худшем случае;  
- память: `O(k)`, где `k` — количество найденных множителей.

Такое решение считается хорошим и стандартным для данной задачи, потому что проверка делителей идет только до квадратного корня, а не до самого числа.
