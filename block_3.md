### 1. Задание 1. Абитуриенты

Есть таблица `examination` с полями:
- `id` — идентификатор абитуриента;
- `scores` — количество баллов за дополнительное вступительное испытание.

Нужно построить общий рейтинг абитуриентов по убыванию количества баллов.

Для этого удобно использовать оконную функцию `RANK()`.  
Она присваивает одинаковый ранг абитуриентам с одинаковым количеством баллов и формирует позицию в общем рейтинге.

Запрос:

    SELECT
        id,
        scores,
        RANK() OVER (ORDER BY scores DESC) AS rating_position
    FROM examination;

Если нужна именно последовательная нумерация без пропусков при одинаковых баллах, можно использовать `DENSE_RANK()`:

    SELECT
        id,
        scores,
        DENSE_RANK() OVER (ORDER BY scores DESC) AS rating_position
    FROM examination;

В рамках обычного рейтинга чаще используют `RANK()`, так как при равных баллах участники делят одно место.

### 2. Задание 2. FULL JOIN

Есть две таблицы:
- первая содержит `30` строк;
- вторая содержит `20` строк.

Нужно определить возможный диапазон количества строк после `FULL JOIN`.

#### Минимальное количество строк

Минимум будет, если все 20 строк второй таблицы найдут совпадения среди строк первой таблицы.  
Тогда в результате останутся все строки первой таблицы, то есть:

`30`

#### Максимальное количество строк

Максимум будет, если ни один ключ не совпадет.  
Тогда в результат попадут все строки из первой и все строки из второй таблицы:

`30 + 20 = 50`

**Ответ:** минимально `30` и максимально `50` строк.

### 3. Задание 3. Покупки

Даны таблицы `account` и `transaction`.

Нужно вывести `ID` клиентов, которые за последний месяц по всем своим счетам совершили покупок меньше, чем на `5000` рублей.

По условию нельзя использовать:
- подзапросы;
- оконные функции.

Логика решения:
- соединяем счета с транзакциями;
- берем только транзакции за последний месяц;
- оставляем только операции покупок;
- группируем по клиенту;
- считаем сумму покупок по всем счетам клиента;
- оставляем только тех, у кого сумма меньше `5000`.

Запрос:

    SELECT
        a.client_id
    FROM account a
    JOIN transaction t
        ON t.account_id = a.id
    WHERE t.transaction_date >= CURRENT_DATE - INTERVAL '1 month'
      AND t.type = 'BUY'
    GROUP BY a.client_id
    HAVING SUM(t.amount) < 5000;

Если в конкретной СУБД нельзя использовать `INTERVAL '1 month'`, то выражение для даты можно адаптировать под нужный диалект.

Например:
- в PostgreSQL подходит текущий вариант;
- в MySQL можно использовать `CURRENT_DATE - INTERVAL 1 MONTH`;
- в Oracle обычно используют `ADD_MONTHS(TRUNC(SYSDATE), -1)`.

---

## 2. Краткие ответы

1. Абитуриенты — запрос через `RANK() OVER (ORDER BY scores DESC)`.  
2. FULL JOIN — минимально `30`, максимально `50` строк.  
3. Покупки — решение через `JOIN`, `GROUP BY`, `HAVING` без подзапросов и оконных функций.
